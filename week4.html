Week 4: N-FINDR Algorithm Implementation

Exploring endmember extraction in hyperspectral imagery using the N-FINDR algorithm

Week 4 Progress: N-FINDR Implementation

This week I focused on studying, understanding, and implementing the N-FINDR algorithm for endmember extraction in hyperspectral imagery, with specific application to the Urban_6 dataset.

Understanding N-FINDR: The Concept

The N-FINDR algorithm is a geometric approach to endmember extraction that works on the principle of finding the maximum volume simplex in the data cloud. The key concepts behind the algorithm are:

  • Endmembers are pure spectral signatures that represent distinct materials in the scene
  • A simplex is a geometric structure that generalizes a triangle to arbitrary dimensions
  • The volume of a simplex formed by pure endmembers will be larger than any simplex containing mixed pixels
  • By maximizing the simplex volume, we can identify the purest pixels in the dataset
  • The algorithm performs dimensionality reduction through PCA to reduce computational complexity

N-FINDR Algorithm Steps:

  1. Reduce dimensionality of the hyperspectral data to p-1 using PCA, where p is the number of endmembers to extract
  2. Randomly initialize a set of p pixels as initial endmembers
  3. Calculate the volume of the simplex formed by these endmembers
  4. For each endmember position, iteratively replace it with each pixel in the dataset and recalculate the volume
  5. If the new volume is larger, keep the replacement; otherwise, retain the original endmember
  6. Repeat until no replacements occur or maximum iterations reached
  7. Project endmembers back to the original spectral space

N-FINDR Implementation

I implemented the N-FINDR algorithm from scratch using Python, NumPy, and scikit-learn. Here's the full implementation:

import numpy as np
from sklearn.decomposition import PCA
import math

# Function to calculate the volume of a simplex
def volume_simplex(A):
    if A.shape[1] <= A.shape[0]:
        # Create matrix for volume calculation: differences from first point + row of ones
        mat = np.vstack((A[:,1:] - A[:,:1], np.ones((1, A.shape[1]-1))))
        mat = mat.T  # shape (p-1, p-1)
        
        # Calculate volume using determinant
        return np.abs(np.linalg.det(mat)) / math.factorial(A.shape[1]-1)
    else:
        return 0

# Main N-FINDR algorithm
def run_nfindr(X, p, max_iter=5):
    # Reduce dimensionality to (p-1) using PCA
    pca = PCA(n_components=p-1)
    X_reduced = pca.fit_transform(X.T).T  # (p-1, N)
    
    # Initialize with random endmember selection
    N = X_reduced.shape[1]
    idx = np.random.choice(N, p, replace=False)
    E = X_reduced[:, idx]
    V = volume_simplex(E)
    
    # Iterative process to maximize simplex volume
    for it in range(max_iter):
        for i in range(p):
            for j in range(N):
                # Try replacing endmember i with pixel j
                E_tmp = E.copy()
                E_tmp[:, i] = X_reduced[:, j]
                V_tmp = volume_simplex(E_tmp)
                
                # Keep replacement if volume increases
                if V_tmp > V:
                    V = V_tmp
                    E[:, i] = X_reduced[:, j]
    
    # Project endmembers back to original spectral space
    endmembers_original_space = pca.inverse_transform(E.T).T
    
    return endmembers_original_space

Insights & Learnings

Through this implementation and experimentation, I gained several key insights about the N-FINDR algorithm:

  • The algorithm's performance is sensitive to initial endmember selection
  • Multiple runs with different initializations can help ensure more reliable results
  • Dimensionality reduction is crucial for computational efficiency, but must be done carefully to preserve essential spectral information
  • The volume calculation is the most mathematically intensive part of the algorithm
  • The algorithm is well-suited for scenes with distinct material types, such as urban environments
